      1. Attributes of Admin Table

        1. fname: The columns which are represents first name of user.

        2. lname: The columns which are represents last name of user.

        3. phone: The column which are represents telephone number of user.

        4. gender: The column which are represents or store gender of user like female and male.

        5. martial_status: The column which are represents martial status of user like single,married,divorced,…..

        6. DoB: The column which are represents Date of Birth of user.

        7. Email: The column which are represents Email of given user.

        8. Password: The column which are represents password of the user.

      2. Attributes of Product Table

        1. product_id: The column which are represents id of product which is unique value called PRIMARY. It is set to auto-increment (AUTO_INCREMENT).
          AUTO_INCREMENT means that the database will automatically generate a unique ID for each new record, typically incrementing the previous value of Product table.

        2. product_name: The column which are represents the name of the product like Prinnter.


        3. price: The column which are represents the price of the product.

        4. stock_quantity: The columns which are represents the quantity stock of the products.

      3. Attributes of Payment Information Table

        1. payment_id: The column which are represents id of payment which is unique value called PRIMARY KEY. It is set to auto-increment (AUTO_INCREMENT).
          AUTO_INCREMENT means that the database will automatically generate a unique ID for each new record, typically incrementing the previous value of payment table.


        2. order_id: The column which are represents id of product which is references to the orders table value called FOREIGN KEY. It is a column of orders table which has relationship between payment information table.


        3. total_amount: The column which are represents the amount of the payment.


        4. transaction_date: The column which are represents the date of payment amount of customer.


      4. Attributes of Shipping Information Table

        1. shipping_id: The column which are represents id of shipping_information which is unique value called PRIMARY KEY. It is set to auto-increment (AUTO_INCREMENT).
          AUTO_INCREMENT means that the database will automatically generate a unique ID for each new record, typically incrementing the previous value of shipping_information table.


        2. order_id: The column which are represents id of product which is references to the product table value called FOREIGN KEY. It is a column of orders table which has relationship between shipping_information table.


        3. shipping_address: The column which are represents the address of the shipping information.


        4. shipping_date: The column which are represents the date of being information shipped.


      5. Attributes of Customer Table

        1. customer_id: The column which are represents id of Customer which is unique value called PRIMARY KEY. It is set to auto-increment (AUTO_INCREMENT).
          AUTO_INCREMENT means that the database will automatically generate a unique ID for each new record, typically incrementing the previous value of Customer table.


        2. first_name: The column which are represents the first name of the customer


        3. last_name: The column which are represents the last name of the customer.


        4. email: The column which are represents the email of customer.


        5. phone: The column which are represents telephone number of customer.


        6. address: The column which are represents address of customer.


      6. Attributes of Orders Table

      1. order_id: The column which are represents id of Customers which is unique value called PRIMARY KEY. It is set to auto-increment (AUTO_INCREMENT).
        AUTO_INCREMENT means that the database will automatically generate a unique ID for each new record, typically incrementing the previous value of Orders table.


      2. customer_id: The column which are represents id of product which is references to the Customers table value called FOREIGN KEY. It is a column of Customers table which has relationship between Orders table.


      3. order_date: The column which are represents the date of being orders of customer.
